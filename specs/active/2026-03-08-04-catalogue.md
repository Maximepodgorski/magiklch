# OKLCH Generator — 4/5 Palette Data & Catalogue

## Context

Create the static palette data (22 Tailwind v4 + 7 curated) and the catalogue page (`/catalogue`) for browsing them. Each palette is a mini stripe preview card. Click → navigates to generator with that palette's base color. Search + filter to find palettes.

**Spec review decision:** Tailwind palettes store **all 11 shades as static data** (not just base color). The catalogue stripe renders these exact values. Click-through passes base hue+chroma to the generator, which regenerates using our algorithm. A disclaimer notes "Generated from Tailwind base — shades may differ from official Tailwind values."

This avoids the trust problem: the catalogue shows the real Tailwind colors, the generator shows our interpretation.

## User Journey (MANDATORY)

### Primary Journey — Browse & Select

ACTOR: UI designer
GOAL: Browse pre-built palettes and open one in the generator
PRECONDITION: App running with generator page (specs 1-3)

1. User navigates to `/catalogue`
   → System displays grid of palette preview cards (22 Tailwind + 7 curated = 29 total)
   → User sees name, source tag (tailwind/curated), mini color stripe per card
   → Tailwind cards show "Tailwind-inspired" source tag

2. User types "green" in search
   → System filters palettes by name in real-time (case-insensitive)
   → User sees matching palettes (green, emerald, lime, teal, moss)

3. User selects "Tailwind" from source filter
   → System shows only Tailwind-inspired palettes
   → User sees 22 cards

4. User clicks "Emerald" palette card
   → System navigates to `/?h=162.48&c=0.17&name=emerald`
   → User sees 11-shade emerald palette in generator (generated by our algorithm, may differ slightly from official Tailwind)

POSTCONDITION: User is in generator with selected palette

### Error Journeys

E1. Search returns no results
   Trigger: User types "xyznonexistent"
   1. User types query with no matches
      → System shows empty state: "No palettes found"
   2. User clears search
      → System shows all palettes again
   Recovery: Clear search or try different term

### Edge Cases

EC1. Case-insensitive search: "RED" matches "Red"
EC2. Source filter + search combined: Both filters apply simultaneously
EC3. Catalogue page direct URL: `/catalogue` loads all palettes without params
EC4. Stripe too narrow on mobile: Show 7 key shades (50, 100, 300, 500, 700, 900, 950) instead of 11

## Analysis

### Assumptions Challenged

| # | Assumption | Evidence For | Evidence Against | Verdict | Action |
|---|------------|-------------|-----------------|---------|--------|
| 1 | 22 Tailwind families is the correct count | Official docs list 22 | GitHub theme.css has 26 (includes mauve, olive, mist, taupe) | VALID | → 22 core for MVP |
| 2 | Static 11-shade data per Tailwind palette is the correct approach | Avoids trust problem (catalogue shows real values) | More data to maintain (~10-15KB gzipped) | **VALID** (decided in spec review) | → no action |
| 3 | Dynamic import isolates catalogue data from generator bundle | Next.js App Router route splitting | Client components may share chunks | RISKY | → verify with bundle analysis |

### Blind Spots

1. **[Data Accuracy]** Our generator's interpretation of Tailwind base colors will differ from official — but the catalogue stripe shows the real values. Honest framing: "Tailwind-inspired."
2. **[UX]** No palette comparison — user can't see two side by side. Out of scope for MVP.

### Failure Hypotheses

| # | IF | THEN | BECAUSE | Severity | Mitigation |
|---|-----|------|---------|----------|------------|
| FH-1 | User clicks Tailwind "Emerald" in catalogue, lands in generator, sees different shades | User loses trust | Generator uses our algorithm, not Tailwind's exact values | HIGH | Catalogue shows static real values; generator framed as "Generated from Tailwind base". Honest positioning. |
| FH-2 | Catalogue data included in generator bundle | Bundle bloat on / route | Shared chunk between routes | MED | Dynamic import + verify |

### The Real Question

This spec is about data integrity + browse UX. The key insight from spec review: separate what we SHOW (real Tailwind static data) from what we GENERATE (our algorithm's interpretation). Honest framing eliminates the trust problem.
